Sorting Algorithms - Selection Sort

Selection sort is one of three processes we will examine in this instructional module to help us modify and organize our arrays for more convenient reference, along with merge sort and counting sort. It is arguably the simplest of the algorithms we will explain within, so this is a good place to start when studying sorting algorithms. Consider the array below.

We have eight elements in our array. Let’s imagine that we wanted to sort these elements by size from least to greatest. To follow the procedure described by selection sort, we first find the smallest value in our array. In this case, it’s 10. We switch this smallest value’s location with that of the first element in the array. In our case, we switch 10 with 15, resulting in the following array:

The integer 10 is now in its final sorted position. As the smallest number in the array, it needs to be in the first position. Therefore, we can cordon the first position of array off, and consider everything to the left of the red line demonstrated below to be sorted.

Now, in effect, we have two arrays. A sorted one to the left of the red line, and an unsorted one to the right. As long as we keep adding elements to the leftmost array from the rightmost array in the order of least to greatest, then the leftmost array will remain sorted as we eventually exhaust every value in the right array. To demonstrate, let’s take the next step in the algorithm: find the smallest element in the array, and move it to the first position. In this case, we move 15 to the start, and switch its position with 30.

We can expand the red line one step forward, increasing the size of the leftmost array and decreasing the size of the right one

Repeat this process until there are no more elements in the rightmost array, and your leftmost array will be a sorted version of your starting array.

Checking each value of the array repeatedly to find which one is the lowest takes your computer time. Selection sort is far from the fastest method for sorting your arrays, especially when array lengths become increasingly long. Consider the size of the arrays you are intending to sort before deciding if selection sort is the correct algorithm for your program.

If you wanted to implement selection sort in JavaScript, you could do so using two nested for loops. Your outer loop would repeat a number of times equal to the length of the original array, counting down as each successive integer is switched. Because of the simplicity of selection sort, we will display the code for the algorithm in JavaScript below, but for the other, more complicated sorting algorithms, we will not. 

function selectionSort(my_Array) {
  for (int i = 0; i < my_Array.length; i++) {
    int firstRightIndex = i;
    int storage;
    int lowestRightIndex = i;
    for (int j = i + 1; j < my_Array.length; j++) {
      if (my_Array[j] < my_Array[lowestRightIndex]) {
        lowestRightIndex = j;
      }
    }
    if (firstRightIndex !== lowestRightIndex) {
       storage = myArray[firstRightIndex];
       my_Array[firstRightIndex] = my_Array[lowestRightIndex];
       my_Array[lowestRightIndex] = storage;
    }
  }
  return my_Array
}
