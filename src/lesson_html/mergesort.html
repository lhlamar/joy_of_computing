Sorting Algorithms - Merge Sort

Merge sort is one of three processes we will examine in this instructional module to help us modify and organize our arrays for more convenient reference, along with selection sort and counting sort. If selection sort could be considered the least difficult of the three, and counting sort the most difficult, then merge sort might be called a middle level of difficulty for the sorting methods presented in this instructional module. Consider the following array:

int my_Array[] = {70, 50, 30, 10, 20, 40, 60}

Let’s imagine we wanted to sort the values in this array in order from least to greatest. Merge sort offers one solution. It proposes that we recursively separate our array in two until we have a collection of single value arrays with a size equal to the length of the original array. In this first step, we separate our seven-value array into two. Then, with one final separation, we reach our target state: a collection of seven arrays of length one.

We were able to complete this process by separating all of our arrays three times, however, many more steps might be necessary to reach this initial target state if the array being sorted is significantly larger. Note that the number of steps required to join these arrays back into a sorted version of the original form is the same number of steps as it took to disassemble the original array in the previous section. Merge sort is simple, and can easily sort arrays with very large numbers, unlike counting sort. However, the length of time that a computer takes to apply this process over a very long array is significant. Consider the size of your array before making use of merge sort.
